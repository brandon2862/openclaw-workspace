## 学习来源优先级

### P1: ClawHub 技能库
- **优先级:** 最高
- **优势:** 专门为 OpenClaw 设计，集成度高
- **命令:** `clawhub search <keyword>`
- **安装:** `clawhub install <skill-name>`

### P2: GitHub 开源项目
- **优先级:** 高
- **优势:** 代码质量高，社区支持好
- **搜索:** GitHub 搜索相关项目
- **参考:** 学习实现思路

### P3: YouTube/B站视频教程
- **优先级:** 中
- **优势:** 直观易懂，有实际操作
- **搜索:** 搜索相关教程视频
- **学习:** 观看并总结关键步骤

### P4: 其他来源
- **优先级:** 低
- **包括:** 文档、博客、论坛
- **参考:** 辅助理解

## 学习流程

### 1. 识别陌生任务
```
用户请求 → 判断是否熟悉 → 不熟悉 → 启动学习流程
```

### 2. 搜索技能
```bash
# 搜索 ClawHub
clawhub search <keyword>

# 如果没有，搜索 GitHub
# 使用 web_search 工具
```

### 3. 安装技能
```bash
# 安装 ClawHub 技能
clawhub install <skill-name>

# 如果是代码参考，克隆仓库
git clone <repo-url>
```

### 4. 学习知识
- 阅读文档
- 理解原理
- 总结要点

### 5. 创建自定义技能 (如果需要)
```
没有现成技能 → 学习相关知识 → 创建自定义 skill
```

### 6. 组合应用
- 组合多个工具
- 完成任务
- 记录经验

## 学习原则

### 快速学习
- **Token 消耗:** < 总 Token 的 20%
- **学习时间:** < 10 分钟
- **目标:** 够用即可

### 够用即可
- 不需要成为专家
- 能完成任务就行
- 深入学习留到以后

### 固化技能
- 重复使用 3 次以上 → 考虑固化
- 解决通用问题 → 可以标准化
- 有长期价值 → 保留到技能库

---

## 陌生任务识别

### 判断标准
1. **从未接触过** - 完全没有相关知识
2. **接触过但不熟悉** - 有概念但不知道怎么做
3. **技术栈不同** - 用过类似技术但具体不同
4. **版本差异** - 用过旧版本，新版本不熟悉

### 应对策略
| 陌生程度 | 策略 | 时间限制 |
|---------|------|---------|
| 完全陌生 | 先搜索 ClawHub | 5 分钟 |
| 有点熟悉 | 直接尝试 + 搜索 | 3 分钟 |
| 技术栈不同 | 参考类似实现 | 5 分钟 |
| 版本差异 | 查看文档 | 2 分钟 |

## 技能固化标准

### 可以固化
- ✅ 重复使用 3 次以上
- ✅ 解决通用问题
- ✅ 可以被标准化
- ✅ 有长期价值

### 不需要固化
- ❌ 只用一次
- ❌ 特定场景专用
- ❌ 太简单不值得
- ❌ 已有现成技能
