## 核心概念

**大脑负责思考，肌肉负责执行。**

---

## 架构概述

```
┌─────────────────────────────────────────┐
│              大脑 (Brain)               │
│  ┌─────────────────────────────────┐  │
│  │         思考与决策              │  │
│  │  - 分析任务                     │  │
│  │  - 制定计划                     │  │
│  │  - 监督执行                     │  │
│  │  - 质量控制                     │  │
│  └─────────────────────────────────┘  │
└─────────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────┐
│              肌肉 (Muscle)              │
│  ┌─────────────────────────────────┐  │
│  │         执行与操作              │  │
│  │  - 运行命令                     │  │
│  │  - 调用工具                     │  │
│  │  - 处理数据                     │  │
│  │  - 生成输出                     │  │
│  └─────────────────────────────────┘  │
└─────────────────────────────────────────┘
```

---

## 大脑角色 (Brain)

### 职责
1. **任务分析**
   - 理解用户需求
   - 分解任务步骤
   - 识别难点和风险

2. **计划制定**
   - 制定执行计划
   - 选择合适工具
   - 分配资源

3. **监督执行**
   - 监控进度
   - 检查质量
   - 调整策略

4. **决策制定**
   - 选择最佳方案
   - 权衡利弊
   - 做出决策

### 特点
- **谨慎** - 不轻易行动
- **全面** - 考虑周到
- **灵活** - 随时调整
- **学习** - 从经验中学习

---

## 肌肉角色 (Muscle)

### 职责
1. **命令执行**
   - 运行 shell 命令
   - 执行脚本
   - 调用 API

2. **工具使用**
   - 文件操作
   - 网络请求
   - 数据处理

3. **输出生成**
   - 生成代码
   - 创建文档
   - 制作报告

4. **重复任务**
   - 批量处理
   - 循环操作
   - 自动化流程

### 特点
- **快速** - 高效执行
- **准确** - 精确操作
- **可靠** - 稳定输出
- **可重复** - 一致的结果

---

## 协作模式

### 模式 1: 大脑主导
```
大脑分析 → 大脑决策 → 肌肉执行 → 大脑检查
```
**适用:** 复杂任务、重要决策、首次处理

### 模式 2: 肌肉主导
```
大脑简单指令 → 肌肉快速执行 → 大脑简单确认
```
**适用:** 简单任务、重复操作、标准化流程

### 模式 3: 协同工作
```
大脑规划 → 肌肉执行 → 大脑监控 → 调整 → 继续执行
```
**适用:** 中等复杂度、需要监督的任务

## 质量控制

### 大脑检查点
1. **执行前** - 计划是否合理？
2. **执行中** - 进度是否正常？
3. **执行后** - 结果是否符合预期？

### 肌肉反馈
1. **成功** - 任务完成，返回结果
2. **失败** - 报告错误，请求指导
3. **不确定** - 暂停，等待确认
