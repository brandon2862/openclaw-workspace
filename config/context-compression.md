- **版本:** 1.0

- **最后更新:** 2026-04-06

---

## 触发条件

### 自动触发
- 对话超过 **10轮**
- 上下文超过 **5K tokens**
- 会话时长超过 **30分钟**

### 手动触发
- 用户要求压缩
- 用户说"总结一下"
- 用户说"清理上下文"

---

## 压缩规则

### 1. 删除冗余
- 重复的内容只保留一次
- 删除多余的空行
- 合并相似的段落

### 2. 简化礼貌用语
- "好的，没问题" → "好的"
- "我来帮你处理" → "处理中"
- "让我想想" → 直接给出答案

### 3. 转换为Markdown结构
- 长段落 → 列表
- 无序内容 → 有序列表
- 重要信息 → **粗体**

### 4. 提取摘要
- 多轮对话 → 一句话总结
- 长文档 → 关键点列表
- 复杂任务 → 进度状态

---

## 压缩策略

### 保留内容 (优先级高)
1. **用户指令** - 明确的任务要求
2. **关键决策** - 重要的决定和原因
3. **代码片段** - 正在处理的代码
4. **错误信息** - 问题和解决方案
5. **文件路径** - 重要的文件位置

### 压缩内容 (优先级中)
1. **对话寒暄** - 问候和确认
2. **详细解释** - 简化为要点
3. **重复确认** - 合并为一次
4. **中间步骤** - 只保留结果

### 删除内容 (优先级低)
1. **已过时的信息** - 不再相关的内容
2. **完全重复的内容** - 多次出现的相同信息
3. **调试日志** - 详细的调试输出
4. **临时数据** - 中间计算结果

## 压缩模板

### 会话摘要格式
```markdown
## 会话摘要
- **时间:** {start_time} - {end_time}
- **任务:** {main_task}
- **状态:** {status}
- **关键决策:** {decisions}
- **待办:** {todos}
```

### 任务进度格式
```markdown
## 任务进度
- [x] 已完成: {task1}
- [x] 已完成: {task2}
- [ ] 进行中: {task3}
- [ ] 待开始: {task4}
```
